# DAPM Versioning

DAPM follows Semantic Versioning.

Version Format:

MAJOR.MINOR.PATCH

Example:

v1.2.3

---

## Major Version

Incremented when fundamental changes are made to the methodology.

Examples:

- Phase redesign
- Workflow restructuring
- Breaking template changes

---

## Minor Version

Incremented when new capabilities are introduced without breaking compatibility.

Examples:

- New templates
- New checklists
- New principles
- Additional documentation

---

## Patch Version

Incremented for corrections and improvements.

Examples:

- Typographical fixes
- Clarified terminology
- Improved wording
- Minor template refinements

---

## Current Version

DAPM v0.1.0

Status: Experimental

The methodology will remain experimental until it has been validated through multiple real-world case studies.
